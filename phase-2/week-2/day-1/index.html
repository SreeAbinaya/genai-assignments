HOW IT WORKS

How To Use

  1. Copy your user story or acceptance criteria into the chat on the right.
  2. Ask for positive, negative, boundary, and validation test cases in a table.
  3. Review the generated output, then copy or export it for your QA workflow.
PROMPT SMARTER

Prompting Tips

  • Include domain, role, business rule, and expected outcome in each story.
  • Ask for columns like Test ID, Scenario, Preconditions, Steps, and Expected Result.
  • Mention if you want API, UI, regression, or edge-case focused coverage.
OUTPUT STYLE

Expected Output

The best results come when you explicitly request a tabular testcase set. Example: “Generate functional, negative, and boundary test cases in a table for the user stories below.”

You can then copy the response or export it into an Excel-friendly format from the chat tools.
